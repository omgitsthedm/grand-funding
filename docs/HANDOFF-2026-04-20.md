## 4. Technical Architecture

### 4.1 What It Is

- **Static HTML/CSS/JS** — zero build step, no framework, no bundler
- **Netlify hosting** — CDN edge delivery globally
- **GitHub repo** → `omgitsthedm/grand-funding` → manual deploy (auto-deploy broken — see § 7.1)

### 4.2 CSS Architecture

| File | Role | Do / Don't |
|------|------|------------|
| `premium-system.css` | Design token source of truth | READ ONLY — never edit without David's approval |
| `styles-v2.css` | Legacy base layer | Don't add to; only prune from |
| `blog.css` | Blog-only deltas | OK to edit |
| `products.css` | Products-only deltas | OK to edit |
| `about.css` | About-only deltas | OK to edit |
| `apply.css` | Apply form deltas | OK to edit |
| `lp.css` | Landing page styles | OK to edit |
| `compliance.css` | Compliance page | OK to edit |
| `premium.css` | **DEPRECATED** — kept for reference only | Never link in HTML |

**Inline `<style>` blocks in HTML** are used for:
- Above-fold critical CSS
- Font `@font-face` declarations
- Page-specific one-offs (marked with `/* === ... === */`)

### 4.3 Font System

**Font:** DM Sans Variable (`/fonts/dm-sans-var.woff2`, 25KB subsetted)

This replaced Poppins + Inter (430KB combined) in the 2026-04-19 font sprint.

```css
@font-face {
  font-family: 'DM Sans';
  src: url('/fonts/dm-sans-var.woff2') format('woff2-variations');
  font-weight: 100 900;
  font-style: normal;
  font-display: swap;
}
body { font-family: 'DM Sans', system-ui, -apple-system, sans-serif; }
```

**Critical:** Every page's `<body>` font-family must include `'DM Sans'` as the first entry. If a page falls back to `system-ui` alone, the brand feel is lost. If it falls through to a serif, the page is broken.

### 4.4 JavaScript

| File | Role |
|------|------|
| `script.js` | Nav, scroll effects, reveal animations, back-to-top |
| `premium.js` | Premium section scroll effects |
| `consent.js` | Cookie consent banner |

All JS is `defer`-loaded. Never use `async` for these — execution order matters for the nav drawer.

**GA4 pattern:** Deferred via `requestIdleCallback(4000)` + first user interaction. Consent defaults + dataLayer setup remain synchronous (legal requirement). Page view event is queued immediately and replays on load.

```js
// Bot detection gate in every JS file
if (navigator.webdriver) return;
```

### Phase 6 — Critical Bug Fixes (2026-04-20, this session)

**Bug 1: `<noscript>`-wrapped stylesheets (CRITICAL)**
- All 40 HTML pages had `styles-v2.css` wrapped in `<noscript>` tags:
  ```html
  <noscript><link rel="stylesheet" href="/styles-v2.css?v=20260428"></noscript>
  ```
- `<noscript>` content only renders when JavaScript is **disabled**. Every real browser has JS enabled. The stylesheet silently never loaded on any page except index.html (which had more inline critical CSS).
- This was a remnant from an earlier async-CSS-loading experiment: the async preload was removed, but only the noscript fallback remained.
- Fix: Python script unwrapped noscript from all 40 pages, restoring blocking stylesheet.
- Commit: `e423378`

**Bug 2: Font-family cascade broken after DM Sans migration**
- The DM Sans swap removed `@font-face` declarations for Poppins and Inter
- `styles-v2.css` body rule and several page-specific stylesheets still referenced `'Poppins'` and `'Inter'`
- Browser fell through the entire fallback stack to Times New Roman (browser serif default)
- Visible on all non-home pages — text rendered in Times New Roman
- Fix: Comprehensive sweep — updated `styles-v2.css` body rule, `styles.css`, and all remaining inline CSS blocks
- Commits: `612e5d1`, `e7e3d2e`

## 7. Known Issues & Technical Debt

### 7.1 Netlify Auto-Deploy Is Broken (CRITICAL)
Auto-deploy from GitHub is not functional. `commit_ref: None` in Netlify — the GitHub → Netlify webhook is broken.

**Workaround (every deploy):**
```bash
cd "/Users/davidmarsh/Desktop/LiFi NYC/Clients/Grand Funding/grand-funding"
netlify deploy --prod --dir=.
```

**Do NOT assume a git push deploys the site.** It doesn't. Always run the manual deploy command after pushing.

To fix permanently: Go to Netlify dashboard → Site settings → Build & deploy → Link to Git → Re-authorize GitHub connection.

### 7.2 `content-visibility: auto` Caution
`content-visibility: auto` was added to `main > section:not(.hero):not(.trust-strip)` for scroll performance. On mobile Safari, this can create 600px phantom whitespace if the `contain-intrinsic-size` is wrong. It was removed once already (commit history shows this). If phantom whitespace appears below fold on Safari, this is the culprit — remove the rule from `styles-v2.css`.

### 7.3 Blog Post Schema
Blog posts have basic structured data but not full Article schema with `author`, `dateModified`, `image`. This is an SEO gap for Google's rich results.

### 7.4 `sitemap.xml` Not Auto-Generated
The sitemap was hand-edited. When new pages are added, `sitemap.xml` must be updated manually. There is no build script for this.

### 7.5 Netlify Forms Not Verified
The contact and apply forms use Netlify Forms. Verify that submissions are reaching the Netlify dashboard and optionally forwarding to Logan's email via Netlify → Notifications.
